
一、后端代码开发规范
本项目所有后端代码采用go-zero框架编写，所以应遵循go-zero开发流程
1、编写接口第一步编写后缀为.api的接口文件，需定义分组开发，例如：
@server (
	group:      "app"
	prefix:     "/api/admin"
	jwt:        Auth
	middleware: AdminAuthMiddleware
)
service power-api {
    <!-- 接口列表 -->
}
2、在根据编写的api文件执行goctl命令，通常会在后端根目录下定义好Makefile文件，执行该Makefile文件命令中的goctl命令即可生成对应的router、handler以及logic文件
例如：管理后台的make gen, 执行的实际就是： goctl api  go  --api .\api\admin.api --home .\common\template\   --dir .  --style gozero

3、此时生成的logic文件都是接口模板文件，只需要删除其中的todo说明，并在其下方完成接口的开发即可

4、开发的配置文件是后缀为yaml的文件，所有数据库、Redis相关的文件均在其中配置， 本地开发mysql账号和密码都是root

5、配置以及所有初始化的操作都需要在svc/servicecontext.go中完成，定义的中间件也需要在此文件中完成初始化
